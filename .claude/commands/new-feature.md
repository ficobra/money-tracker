# New Feature Planning

Before implementing any new feature, follow these steps:

1. Ask the user to describe the feature in detail
2. Create a written plan including:
   - Which files will be modified
   - Database changes needed (new tables, columns)
   - UI changes needed (new views, modifications)
   - Potential breaking changes or side effects
3. Present the plan clearly and wait for explicit approval
4. Only start implementation after user confirms the plan

Do not write any code until the plan is approved.
